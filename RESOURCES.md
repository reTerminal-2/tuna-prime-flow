Resources

•	Hardware
Desktop computer – The primary device for system development, coding, debugging, and running the development environment. 
Wi-fi – Provides the necessary internet connection for real-time data synchronization, communication between devices, and system updates.
Barcode Scanner – Used to easily read and identify inventory products with complex names.
Receipt Printer – Used to print physical records of transactions.
Azure Server – The cloud infrastructure used to host the Virtual Machine (VM) running the reverse-engineered chatbot and backend AI services.

•	Software
Visual Studio Code – The integrated development environment (IDE) for writing, debugging, and deploying the system’s application. 
React, Vite & TypeScript – The core development platform and language used for building and customizing the web-based TunaFlow interface.
Supabase – The mobile app and web development platform that provides the system’s backend cloud database and real-time features.
C++ – Utilized in the secondary Virtual Machine (VM) backend to support high-performance operations for the reverse-engineered chatbot.
Netlify – Used for cloud hosting, continuous deployment, and serverless Edge Functions for the TunaFlow web application.
Python – Provides the necessary libraries and scripting environment for the reverse-engineered chatbot and AI services.
JavaScript – Used for backend scripting, deployment automation scripts (like FTP uploading), and serverless Edge Function logic.
Node.js – The runtime environment that executes JavaScript code outside the browser, powering development tools and server-side scripts.
JSON – The standard file format and data structure used for configuration, data interchange between the frontend and backend, and storing product models.
Tailwind CSS – A utility-first CSS framework used for building a modern, responsive, and highly customized user interface.
shadcn/ui – A collection of re-usable components built using Radix UI and Tailwind CSS, providing the system's high-quality design and interactive elements.
Docker – Used for containerizing the backend services, ensuring consistent environments across development and production VMs.
Bun – A fast all-in-one JavaScript runtime and package manager used to speed up the development workflow and dependency management.
